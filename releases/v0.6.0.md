# agent-local v0.6.0 — AUDIT R8 remediation

**Date**: 2026-07-01 · **Line**: pre-1.0 · **Upstream audit**:
template_MLOps `docs/audit/AUDIT_R8_STAFF_LEAD.md`

First release cut after the first dual-repo staff/lead audit. The audit's
thesis — *the platform adopted the sibling template's governance philosophy
but not yet its gates, and every discipline that traveled without a gate had
a live finding* — is answered here: all code findings fixed, three new gates
added so none of them can come back silently.

## Highlights

- **The event loop is free again** (R8-01, HIGH): `/dev/message` ran the
  full synchronous multi-LLM loop inside `async def`, blocking every
  concurrent request. Fixed (threadpool `def`) and pinned by an AST-based
  serving contract test — the same D-24 discipline the template enforces.
- **Reflection now buys signal** (R8-03, ADR-009): the reflect station's
  output was discarded; it now feeds generation through a dedicated notes
  channel that can never masquerade as tool evidence for the verifier.
- **One version, everywhere** (R8-04): `core.__version__` is the single
  source; pyproject is dynamic, the API surface imports it, and a new
  4-check coherence gate (`scripts/check_coherence.py`) fails CI on drift —
  the calibrated port of template rule 16.
- **No more silent leaks or false 200s**: 500s return a correlation id
  instead of `str(e)` (R8-02); the WhatsApp stub answers 501 (R8-09).
- **Enforcement parity**: full-surface lint (R8-06), gitleaks in CI +
  pre-commit (R8-12), ratio-based eval gate with non-zero exit on failure
  (R8-07).

## Compatibility

No breaking API changes. `/dev/message` request/response schema unchanged;
error bodies now carry `error_id` instead of raw exception text. The
`/webhook/whatsapp` stub status code changed 200 → 501 (correct per its
documented contract; Phase-1 clients should not have depended on the 200).
